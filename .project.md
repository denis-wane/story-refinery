# Project Context: Story Refinery

_Last updated: 2026-06-09_

## Architecture overview

**Architecture style:** monolith (Next.js full-stack)
**Deployment:** local development initially, potential Vercel deployment later
**Environments:** dev (local only for now)

### Key components
| Component | Purpose | Tech | Location |
|-----------|---------|------|----------|
| Web UI | Dashboard, pipeline visualization, forms | React 19 + Tailwind CSS 4 | `src/app/`, `src/components/` |
| API Routes | Pipeline management, config, Jira proxy | Next.js Route Handlers | `src/app/api/` |
| Pipeline Engine | Step execution, event streaming, state management | TypeScript + Anthropic SDK | `src/lib/pipeline/` |
| Storage | Config, run/step state, reviews | JSON files on filesystem | `src/lib/store.ts` |
| Step Definitions | Agent prompts and pipeline configuration | TypeScript | `src/lib/pipeline/steps.ts` |

### System boundaries
- **In scope:** Web UI, pipeline orchestration, Jira read-only integration, local file I/O, Claude API calls
- **External:** Jira REST API (read-only), Anthropic Claude API, local file system

## Domain glossary

| Term | Definition |
|------|-----------|
| Pipeline | A sequence of agent steps that transforms input into refined output |
| Step | A single agent operation within a pipeline (e.g., Analyze, Decompose) |
| Run | One execution of a pipeline with specific input |
| Review | Human feedback attached to a step, can approve or request changes |
| Review Gate | A checkpoint after a step where the pipeline pauses for human review |
| Generate Mode | Creates stories from raw input (description/idea) |
| Refine Mode | Improves existing stories pulled from Jira or local files |
| Acceptance Criteria (AC) | Testable conditions that define when a story is complete |

## API conventions

### API style
REST (Next.js Route Handlers)

### Request/response patterns
- JSON request/response bodies
- SSE (Server-Sent Events) for real-time pipeline progress
- Standard error format: `{ error: string }`

## Data model (key entities)

| Entity | Key fields | Relationships | Notes |
|--------|-----------|---------------|-------|
| Config | key, value | standalone | Key-value config store |
| PipelineRun | id, mode, status, input | has many Steps | Top-level execution |
| PipelineStep | id, run_id, name, agent, status, input, output | belongs to Run, has many Reviews | Individual agent step |
| Review | id, step_id, run_id, comments, approved | belongs to Step and Run | Human review |

## Project-specific rules for agents
- Dark theme UI — all components use gray-900/950 backgrounds, gray-100/200 text
- Pipeline steps are defined declaratively in `src/lib/pipeline/steps.ts`
- All state persisted to JSON files under `data/` — runs survive server restarts
- Jira integration is read-only — never write to Jira
- Output files from Jira sources use Jira issue key as filename (e.g., `PROJ-123.md`)
